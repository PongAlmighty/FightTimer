# Bitfocus Companion Integration Guide

## API Overview
The Fight Timer provides a REST API endpoint for integration with Bitfocus Companion, allowing remote control of the timer through HTTP requests.

Base URL: `http://your-server:8765/api/timer`

## HTTP Methods

### GET /api/timer
Check API status and get supported actions.

Response:
```json
{
    "status": "success",
    "message": "Timer API endpoint is active",
    "supported_actions": ["start", "stop", "reset"]
}
```

### POST /api/timer
Control the timer with specific actions and parameters.

Request body format:
```json
{
    "action": "start|stop|reset",
    "minutes": 5,  // Optional, default: 5
    "seconds": 0   // Optional, default: 0
}
```

## Setting Up Bitfocus Companion

1. Open Bitfocus Companion
2. Add a new button
3. Select "Generic HTTP/UDP"
4. Configure the button:
   - Type: "REST"
   - URL: "http://your-server:8765/api/timer"
   - Request Type: "POST"
   - Content Type: "application/json"

## Example Button Configurations

### Start Timer Button
```json
{
    "action": "start"
}
```

### Stop Timer Button
```json
{
    "action": "stop"
}
```

### Reset Timer (5 Minutes) Button
```json
{
    "action": "reset",
    "minutes": 5,
    "seconds": 0
}
```

### Reset Timer (3 Minutes) Button
```json
{
    "action": "reset",
    "minutes": 3,
    "seconds": 0
}
```

## Troubleshooting

1. Ensure the Fight Timer server is running on port 8765
2. Verify your network allows connections to the server
3. Test the API using curl before configuring Companion:
   ```bash
   # Check API status
   curl http://your-server:8765/api/timer
   
   # Start timer
   curl -X POST http://your-server:8765/api/timer \
        -H "Content-Type: application/json" \
        -d '{"action":"start"}'
   ```

For additional support or questions, please open an issue on the GitHub repository.
